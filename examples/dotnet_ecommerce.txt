Build an E-Commerce Platform with .NET and Blazor

PROJECT OVERVIEW:
Create a modern, full-stack e-commerce platform using .NET 8, Blazor WebAssembly for the frontend, ASP.NET Core Web API for the backend, and SQL Server for data persistence. Include admin panel, payment processing, and order management.

CORE FEATURES:

1. Customer Frontend (Blazor WebAssembly)
   - Product catalog with search and filtering
   - Product detail pages with images and reviews
   - Shopping cart management
   - Checkout process with multiple payment options
   - User account management
   - Order history and tracking
   - Wishlist functionality
   - Product reviews and ratings

2. Admin Panel (Blazor Server)
   - Dashboard with sales analytics
   - Product management (CRUD operations)
   - Category and brand management
   - Order management and fulfillment
   - Customer management
   - Inventory tracking
   - Sales reports and analytics
   - Discount and coupon management

3. Authentication & Authorization
   - ASP.NET Core Identity
   - JWT token authentication
   - Role-based access control (Customer, Admin, Manager)
   - OAuth providers (Google, Microsoft, Facebook)
   - Two-factor authentication (2FA)
   - Email verification
   - Password reset functionality

4. Product Management
   - Product variants (size, color, etc.)
   - Product categories and subcategories
   - Product images (multiple per product)
   - Stock management
   - Price management with discounts
   - Product attributes and specifications
   - Related products

5. Shopping Cart & Checkout
   - Persistent shopping cart
   - Cart totals with tax calculation
   - Shipping options and costs
   - Discount code application
   - Guest checkout option
   - Order summary
   - Email confirmations

6. Payment Processing
   - Stripe integration
   - PayPal integration
   - Credit card processing
   - Payment history
   - Refund processing
   - Secure payment handling

7. Order Management
   - Order creation and tracking
   - Order status updates (Pending, Processing, Shipped, Delivered)
   - Order history
   - Invoice generation (PDF)
   - Shipping label integration
   - Return and refund management

8. Search & Filtering
   - Full-text search with Azure Search or Elasticsearch
   - Faceted filtering (category, price, brand, rating)
   - Sorting options
   - Search suggestions
   - Recent searches

9. Notifications
   - Email notifications (order confirmation, shipping updates)
   - SignalR for real-time notifications
   - Admin alerts for low stock
   - Customer notifications for wishlist items on sale

DESIGN PATTERNS:

1. Repository Pattern
   - Generic repository for CRUD operations
   - Unit of Work pattern

2. CQRS (Command Query Responsibility Segregation)
   - MediatR for command/query handling
   - Separate read and write models

3. Specification Pattern
   - Flexible query building
   - Reusable business rules

4. Factory Pattern
   - Payment provider factory
   - Notification service factory

5. Strategy Pattern
   - Different payment strategies
   - Shipping cost calculation strategies

FEATURES IMPLEMENTATION:

API Endpoints:
```csharp
// Products
GET    /api/products
GET    /api/products/{id}
POST   /api/products
PUT    /api/products/{id}
DELETE /api/products/{id}
GET    /api/products/search?q={query}
GET    /api/products/category/{categoryId}

// Orders
GET    /api/orders
GET    /api/orders/{id}
POST   /api/orders
PUT    /api/orders/{id}/status
GET    /api/orders/customer/{customerId}

// Cart
GET    /api/cart
POST   /api/cart/items
PUT    /api/cart/items/{id}
DELETE /api/cart/items/{id}
POST   /api/cart/checkout

// Auth
POST   /api/auth/register
POST   /api/auth/login
POST   /api/auth/refresh-token
POST   /api/auth/forgot-password
POST   /api/auth/reset-password
```
